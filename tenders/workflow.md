# Полный сценарий работы с тендерами

## Этап 1 — Поиск тендеров
**Скилл:** `tenders-moldova`
**Cron:** пн-пт 8:05, 12:05, 16:05

Поиск на achizitii.md тремя режимами:
- **A — CPV-коды:** 35000000-4 + 13 доп. кодов (+ родительские)
- **B — Ключевые слова:** румынские термины безопасности
- **C — Организации:** 72 компании из топ-550 налогоплательщиков

Проверка актуальности: публикация ≤ 60 дней, дедлайн > сейчас.

Результат: таблица найденных тендеров → в Telegram.

## Этап 1.5 — Отбор тендера
Ты говоришь «этот берём» или я на основе критериев предлагаю, ты одобряешь.

## Этап 2 — Создание папки тендера
**До скачивания любых документов.**

Создаю в `~/wiki/tenders/`:
```
tenders/{ID}_заказчик/
├── info.md          ← карточка (сумма, дедлайн, статус 🔵 Новый)
├── checklist.md     ← будет заполнен после получения требований
├── original/        ← документы от заказчика
└── our/             ← наши документы
```
ID — число из ссылки achizitii.md: `/tender/{ID}/`.

Статусы: 🔵 Новый → 🟡 В работе → 🟢 Отправлен → 🔴 Просрочен

## Этап 3 — Экспертиза (tender-expert-md)
1. Скачиваю Anunț de participare из MTender
2. Извлекаю полный перечень требуемых документов
3. Проверяю каждое требование на соответствие LP131/2015:
   - Дискриминационные требования (ст. 23)
   - Чрезмерные требования к опыту (ст. 17)
   - Сокращённые сроки (ст. 45)
   - И т.д.
4. Вывожу чек-лист:
   ```
   | № | Документ | Статус |
   |---|----------|--------|
   | 1 | Cerere de participare | ❌ Нужно заполнить |
   | 2 | Specificații tehnice | ❌ Нужно заполнить |
   | 3 | Certificat de înregistrare | ✅ Есть |
   ```
5. Ты утверждаешь → переходим к заполнению
6. Обновляю статус в info.md: 🟡 В работе

## Этап 4 — Заполнение документов
1. Беру пустые формы (DOCX) из `~/wiki/company/templates/forms/` или от тебя
2. Заполняю через python-docx:
   - LISNIC ALARM S.R.L., IDNO 1007600024288
   - IBAN, банк Moldova-Agroindbank
   - Номер тендера, заказчик, дата
   - Технические спецификации, цены
   - Опыт, лицензии
3. Сохраняю как `вар1` → `вар2` → `FINAL`
4. Старые версии не удаляю без твоего подтверждения
5. Файлы кладу в `tenders/{ID}_заказчик/our/`
6. Выгружаю на Google Drive: `wiki-backups/tenders/{ID}_заказчик/`

## Этап 4.5 — Поиск недостающих сертификатов
После чек-листа проверяю наличие всех сертификатов:
- **Сертификаты** → alarm.md (раздел «Сертификаты»), сайты производителей
- **Даташиты** → satel.pl (`/api/product/leaflet/{PRODUCT}/en`), bcs.pl, westerndigital.com
- **CE-декларации** → сайты производителей / Declaration of Conformity
- **Лицензии** → проверяю сроки в `~/wiki/company/certificates/`

Скачанные файлы → `tenders/{ID}_заказчик/our/` + `~/wiki/company/equipment/{BRAND}/`

## Этап 5 — Проверка и сдача

### Перед отправкой
Проверяю каждый документ на 4 обязательных поля:
| Поле | Пример |
|------|--------|
| Заказчик | Către: Î.M. „IMC” S.R.L. |
| Номер тендера | Nr. procedurii: ocds-b3wdp1-MD-... |
| Дата | Data: 14.07.2026 |
| Компания | Ofertantul: LISNIC ALARM S.R.L. |

### Финальная проверка
Ты проверяешь итоговый пакет, я меняю статус → 🟢 Отправлен.
